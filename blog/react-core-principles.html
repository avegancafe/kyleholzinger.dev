Guidelines for writing maintainable react applications

After working with react for quite a few years (I think 8?), I've noticed some patterns that I tend to stick with. They're not super hard and fast rules, but more so guidelines that I try to use as a first instinct when using react. They may be applicable to other libraries/frameworks, and I'm sure some of them are informed by my work in other environments, but these are specifically targeted at react.

  • Avoid useEffect
  • Use custom hooks to isolate complex behavior
  • Compound components can clean up component APIs

There are probably some others that I'm missing, but I think these are the big three. Let's get into them.

Avoid useEffect

One of the things I've noticed when working in react codebases is that it's very easy to fall into the trap of using useEffect for a wide variety of scenarios. E.g. updating a computed property, async fetching some data, etc. But IMO, this is a trap. As much as physically possible, I try to push these "effects" into something like a callback or event handler or something of the sort.
